linux 的基本操作指令：
欢迎来学习~~

一：find 命令

1:查找文件（根据文件格式进行查找）查找/home 下的以.txt结尾的文件：find /home -name “*.txt”

2:递归查找所有的文件（当前目录及子目录下查找所有以.txt和.pdf结尾的文件,-o 的意思为或者 ）: find . -name “.txt” -o -name “.pdf”

3: 查找 /home 下不是以 .txt结尾的文件：find /home ! -name “.txt”

4:根据文件类型进行查找： -type 为类型参数

  • f 表示的是文件
  • d 表示的是目录
  • c 表示的是字符设备
  • b 表示的是块设备
  • s 表示的是套接字
  • i 表示的是链接

    5: 查找/home 下的子目录：find /home -type d

    6: 根据文件目录的层数进行查找：find -maxdepth+层数 按照最大的层数来查找，-mindepth+层数 按最小的层数来查找

    7：查找/home 下目录最大层数为2层的文件: find /home -maxdepth - -type f

    二： 查看一个程序是否运行

    1: 查看所有的tomcat的进程：ps -ef|grep tomcat

    2:终止线程19979的进程 终止线程：kill -9 19979

    3：批量终止java进程:pkill -9 java

    三： 查看文件

    1:产看文件，包括隐藏文件：ls -al

    2:查看目录：pwd

    3:创建目录：mkdir fold

    4:递归删除目录：rm -rf fold

    5:移动文件：mv /tmp /tofold

    6:重命名文件：mv file1 file2

四： 复制文件

1:复制当前文件到目标地址 ：cp source dest

2:复制当前目录”test/“下的所有的文件到新的目录 newtest下 ：cp –r test/ newtest

3：远程拷贝（从远处复制目录到本地） ：scp -r root@192.168.120.204:/opt/soft/mongodb /opt/soft/

4：远程拷贝（从远处复制文件到本地目录） ：scp root@192.168.120.204:/opt/soft/nginx-0.5.38.tar.gz /opt/soft/

5：远程拷贝（从本地服务器复制到远程服务器：） ：scp local_file remote_username@remote_ip:remote_folder

五: 权限

切换用户：su -username

su方式切换是须要输入目标用户的password。而sudo仅仅须要输入自己的password，所以sudo能够保护目标用户的password不外流的。当帮root管理系统的时候，su是直接将root全部权利交给用户。而sudo能够更好分工，仅仅要配置好/etc/sudoers，这样sudo能够保护系统更安全，并且分工明白，有条不紊。

sudo -s 切换root权限 ,sudo su 和 sudo -s都是切换到root用户，不同的是：

  • sudo su 环境用的是目标用户(root)的环境
  • sudo -s 环境用的是当前用户本身的环境

    例如：sudo head head -n 30000 /etc/sudoers

    修改文件权限: chmod 777 file.java

    六：文件

    1:压缩文件 tar -czf test.tar.gz /test1 /test2

    2:压缩成zip 的文件 ：zip -r test.zip /test

    3:列出压缩文件的列表 ：tar tzf test.tar.gz

    4:解压gz文件 ：tar -zxvf test.tar.gz

    5:解压zip文件 ：unzip test.zip

七：查看日志

查看头10行的：head -n 10 example.txt

查看后10行的：tail -n 10 example.txt

查看日志最新的10行: tail -f example.log

查看日志的后10000行,关键字是 Exception 的前后50行的日志：tail -n 10000 aa.log|grep -C 50 ‘Exception’

根据行号查找对应的数据1：cat -n aa.log |grep ‘aa’

根据行号查找对应的数据2：cat -n aa.log |tail -n +3586|head -n 100

less查看日志：less a.log

八 系统配置

在LINUX 系统中的配置文件的路径为：/etc/profile

使用 cat/etc/profile 来查看配置文件的内容，profile 配置文件的的信息：

  • export JAVA_HOME =/opt/jdk
  • export CLASSPATH=.
  • export PATH=/opt/jdk/bin:$PATH
    JAVA_HOME 指向JAVA JDK 的安装目录，通知某些软件如何找到JDK的安装目录，CLASS_PATH 表示类的搜索路径，简单的可以点.PATH 指向的JAK的bin 的目录 javac 和java等命令安装在此目录中

    九查看端口号

    查看端口的状态 ：lsof -i:8080

    查看哪些端口被打开：netstat -anp

    关闭端口号(通过iptables工具将该端口禁掉，)1：sudo iptables -A INPUT -p tcp –dport $PORT -j DROP” , sudo iptables -A OUTPUT -p tcp –dport $PORT -j DROP”

    关闭端口号(关掉对应的应用程序，则端口就自然关闭了)2：kill -9 PID” (PID：进程号)

kill -sig pid,这里的 sig 可以是信号对应的数字，也可以是信号名，比如你如果用 kill -9 pid 实际是发 9号信号给进城，9对应的信号名是 KILL。所以 kill -9 等价于 kill -KILL pid。 常用的几个信号是 :

端口 数字 描述
INT 2 这个就是你在bash下面用Ctrl+C 来结束一个程序时，bash会向进程发送这个信号，默认的，进程收到这个程序会结束。 你可以用 kill -INT pid 来发这个信号。
QUIT 3 这个是你在bash下用 Ctrl+\ 来结束程序时，发的信号，进程默认受到这个信号后也是结束
KILL 9 这个信号之所以被称为“强杀”，就是因为无法改变进程收到这个信号后所执行的动作，进程只能退出。(前面说的两个信号，虽然默认是退出，但是应用程序自己可以通过signal系统调用来修改成其他动作，比如忽略那两个信号等动作)

打开端口号,1. 开放端口命令：：1： /sbin/iptables -I INPUT -p tcp –dport 8080 -j ACCEPT 2.保存：/etc/rc.d/init.d/iptables save 3.重启服务：/etc/init.d/iptables restart 4.查看端口是否开放：/sbin/iptables -L -n

直接编辑/etc/sysconfig/iptables文件：1.编辑/etc/sysconfig/iptables文件：vi /etc/sysconfig/iptables, 加入内容并保存：-A RH-Firewall-1-INPUT -m state –state NEW -m tcp -p tcp –dport 8080 -j ACCEPT 2.重启服务：/etc/init.d/iptables restart 3.查看端口是否开放：/sbin/iptables -L -n

十：防火墙

查看防火墙的状态: service iptables status /etc/init.d/iptables status

开启防火墙: service iptables start

关闭防火墙：service iptables stop

暂时关闭防火墙： /etc/init.d/iptables stop

重启iptables : /etc/init.d/iptables restart

获取外网的ip:curl ipecho.net/plain

更改文件夹的权限,路径权限：chown centos.centos /app/jar -R 更改文件夹的权限。路径全线

修改文件夹的读写和执行权限： chmod 777 file

十一：查看系统信息

查看内存信息：free -m

查看系统的日志信息：cat /var/log/messages

设置服务器时间

1:安装ntpdate 工具

2：设置系统时间与网络时间同步： ntpdate cn.pool.ntp.org 或者 ntpdate -u -u ntp1.aliyun.com

3：查看当前时区date -R

4: 方法A,修改设置Linux服务器时区,tzselect

5: 方法B 仅限于RedHat Linux 和 CentOS“timeconfig”

6: 方法C,适用于Debian,dpkg-reconfigure tzdata

7:复制相应的时区文件，替换系统时区文件；或者创建链接文件:cp /usr/share/zoneinfo/$主时区/$次时区 /etc/localtime 例如：cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

8：查看和修改Linux的时间: date”

9：例如：将系统日期设定成2009年11月3日的命令: date -s 11/03/2009

10：将系统时间设定成下午5点55分55秒的命令: date -s 17:55:55

11: 将当前时间和日期写入BIOS，避免重启后失效:hwclock -w

12:不加参数可以直接看到本月月历: cal

十二 ：执行jar包命令

执行jar命令：nohup /app/jdk1.8.0_45/bin/java -jar -Dspring.profiles.active=test /app/jar/ROOT/temp/opti/tamc-service-api.jar >/app/jar/ROOT/temp/tamc-service-api.log 2>&1 &

执行jar 命令： nohup > notify.log java -jar tamc-cloud-notify.jar 2>&1 &

执行jar命令： nohup /app/jdk1.8.0_45/bin/java -jar -Dspring.profiles.active=test /app/jar/ROOT/temp/opti/tamc-service-api.jar >/app/jar/ROOT/temp/tamc-service-api.log 2>&

执行jar命令： app/jdk1.8.0_45/bin/java -jar -Dspring.profiles.active=test /app/jar/ROOT/tamc-service-api.jar >/app/jar/logs/tamc-service-api.log 2>&1 &

执行jar包： 1234是生成的日志文件 nohup java -Xms1024m -Xmx2048m -jar ecn.cron.jar >1234 & tail -f 1234

执行jar包： nohup > notify.log java -jar tamc-cloud-notify.jar 2>&1 &
